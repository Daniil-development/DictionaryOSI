
Бит/Bit
Бит - единица данных физического уровня.
~
Дейтаграмма/Датаграмма/Датаграма/Datagram
Дейтаграмма - единица данных протокола UDP.
~
Интерфейс/Интерфэйс/Interface
Интерфейс - набор формализованных правил, по которым обмениваются информацией сетевые компоненты соседних уровней одного узла. 
~
Кадр/Frame
Кадр - единица данных канального уровня.
~
Канальный уровень/Data Link Layer
Канальный уровень связывает узлы в сеть, обеспечивает надёжную передачу данных через физический канал.
Единица данных канального уровня - кадр.
Основное назначение канального уровня - формирование кадра и отправка его в сеть, приём кадра из сети.
~
MTU/Maximum Transmission Unit
MTU - максимальная единица передачи.
~
OSI/Open Systems Interconnection/Взаимосвязь открытых систем
OSI (Open Systems Interonnection) - Эталонная модель взаимодействия открытых систем. Разработана Международной организацией по стандартизации ISO (International Standarts Organisation) в 1984 г.
Модель OSI/ISO стала международным стандартом для построения сетей различных типов.
~
Открытая система/Open system
В широком смысле:
Открытой системой называется любая система, которая построена в соответствии с открытыми спецификациями (опубликованными, общедоступными, соответствующими стандартам).
В узком смысле:
Открытая система - это система, реализующая стандартный набор функций, поддерживаемая стандартными протоколами и отвечающая требованиям эталонной модели OSI.
~
Пакет/Packet
Пакет - единица данных сетевого уровня.
~
Протокол/Protocol
Протокол - набор формализованных правил, по которым обмениваются информацией сетевые компоненты, лежащие на одном уровне в разных узлах сети.
~
Протокольный блок данных/Protocol Data Unit/PDU
Протокольный блок данных - единица данных в стандартах ISO.
~
Сегмент/Segment
Сегмент - единица данных протокола TCP.
~
Сообщение/Message
Сообщение - единица данных прикладного уровня, это логически завершённая порция данных (например, файл).
~
Спецификация/Specification
Спецификация - формализованное описание аппаратных или программных компонентов, способов их функционирования, взаимодействия с другими компонентами, условий эксплуатации.
~
Стек коммуникационных протоколов/Communication protocol stack
Стек коммуникационных протоколов - иерархический набор протоколов, достаточный для организации взаимодействия узлов в сети.
~
Уровни OSI/OSI levels
Модель OSI включает 7 уровней:
7. Прикладной (Application Layer)
6. Представительный (Presentation Layer)
5. Сеансовый (Session Layer)
4. Транспортный (Transport Layer)
3. Сетевой (Network Layer)
2. Канальный (Data Link Layer)
1. Физический (Physical Layer)
~
Физический уровень/Physical Layer
Физический уровень описывает процесс передачи потока битов по физическим каналам связи между цифровыми устройствами (например, по сетевому кабелю или цифровому территориальному каналу).
Единица данных - бит.
Функции физического уровня реализуются на всех устройствах, подключённых к сети.
Со стороны компьютера функции физического уровня выполняет сетевой адаптер или последовательный порт.
